# Módulo 12 — Modelo Mental Integrado

## Propósito

Sintetizar todos los módulos anteriores en un modelo mental unificado: trazar el flujo completo de ejecución desde que un cliente envía un request hasta que recibe la respuesta, y consolidar los principios clave que gobiernan el rendimiento de un sistema.

## Temas

| # | Archivo | Tema |
|---|---------|------|
| 12.1 | `12.1_Flujo_de_Ejecucion.md` | Cliente → NIC → kernel → CPU → RAM → disco → respuesta |
| 12.2 | `12.2_Principios_Clave.md` | Latencia, throughput, paralelismo, networking, persistencia |

## Dependencias

- Todos los módulos anteriores (01-11)

## Conexión con el sistema

Este módulo cierra el ciclo. No es nuevo contenido técnico sino la integración de todo lo aprendido en un modelo mental que permite razonar sobre cualquier problema de rendimiento, diagnosticar bottlenecks, y tomar decisiones de diseño informadas por la realidad del hardware subyacente.
