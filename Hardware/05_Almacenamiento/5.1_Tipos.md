# 5.1 Tipos de Almacenamiento

## Definición

Los tres tipos principales de almacenamiento persistente son HDD (Hard Disk Drive) con platos magnéticos giratorios, SSD (Solid State Drive) con memoria NAND Flash sin partes móviles, y NVMe (Non-Volatile Memory Express) que es un SSD conectado directamente por PCIe sin el overhead del protocolo SATA/AHCI. La diferencia de rendimiento entre ellos puede ser de 100-1000× dependiendo del patrón de acceso.

## Mecánica interna

### Comparación

```
  ┌──────────────────┬──────────────┬──────────────┬──────────────┐
  │ Característica   │ HDD          │ SSD (SATA)   │ NVMe SSD     │
  ├──────────────────┼──────────────┼──────────────┼──────────────┤
  │ Latencia random  │ 5-10ms       │ 50-100μs     │ 10-20μs      │
  │ Latencia seq.    │ ~5ms+        │ ~50μs        │ ~10μs        │
  │ IOPS random read │ 100-200      │ 50K-100K     │ 500K-1M+     │
  │ IOPS random write│ 100-200      │ 20K-50K      │ 200K-500K    │
  │ BW secuencial    │ 100-200 MB/s │ 500-550 MB/s │ 3-7 GB/s     │
  │ Interfaz         │ SATA         │ SATA         │ PCIe (×4)    │
  │ Queue depth      │ 32 (AHCI)    │ 32 (AHCI)    │ 65535 (NVMe) │
  │ Costo/TB         │ ~$15-25      │ ~$50-80      │ ~$60-100     │
  │ Durabilidad      │ 3-5 años     │ 5-10 años    │ 5-10 años    │
  │ Consumo          │ 5-10W        │ 2-4W         │ 5-8W         │
  └──────────────────┴──────────────┴──────────────┴──────────────┘

  Diferencia clave: IOPS
  Un HDD puede hacer ~150 accesos aleatorios por segundo.
  Un NVMe puede hacer ~1,000,000 accesos aleatorios por segundo.

  Para una base de datos con 10,000 queries/segundo (cada una accediendo
  a ~10 páginas random): HDD necesitaría 100K IOPS → imposible.
  NVMe lo maneja sin problema.
```

### Casos de uso

```
  HDD:
  - Backups y archival (costo/TB imbatible)
  - Cold storage (datos accedidos raramente)
  - Logs de largo plazo
  - NAS para archivos grandes (video, música)

  SSD SATA:
  - Boot drive (OS, aplicaciones)
  - Base de datos de volumen medio
  - VMs con I/O moderado
  - Budget-friendly general purpose

  NVMe:
  - Bases de datos de alto rendimiento (PostgreSQL, MySQL en prod)
  - Cachés de disco (bcache, dm-cache)
  - Build systems (compilación, CI/CD)
  - Aplicaciones latency-sensitive
  - Logging de alta frecuencia

  Regla: si tu workload es IO-bound, el upgrade HDD→NVMe puede
  dar 100-1000× mejora. Es la optimización de hardware más impactante.
```

## Modelo mental

HDD es un tocadiscos: un brazo físico debe moverse al track correcto y esperar a que el plato gire hasta el sector deseado. SSD es un array de memoria Flash: no hay partes móviles, el acceso es electrónico y casi instantáneo. NVMe es un SSD sin el cuello de botella de SATA: conectado directamente al PCIe bus con un protocolo diseñado para paralelismo masivo (65K queues vs 1 queue en SATA).

## Insights de ingeniería

- **El patrón de acceso importa más que el tipo de disco**: un HDD con acceso secuencial (200 MB/s) puede ser aceptable. El mismo HDD con acceso aleatorio (150 IOPS × 4KB = 0.6 MB/s) es inaceptable. La pregunta no es "¿qué disco tengo?" sino "¿cuál es mi patrón de acceso?"
- **Queue depth es el multiplicador oculto de NVMe**: SATA/AHCI soporta 32 comandos en cola. NVMe soporta 65,535 comandos en 65,535 colas. Esto permite que el controlador NVMe reordene y paralelice accesos internamente, explotando el paralelismo de los chips NAND Flash. Por esto NVMe es 10× más rápido que SSD SATA con el mismo tipo de NAND.
