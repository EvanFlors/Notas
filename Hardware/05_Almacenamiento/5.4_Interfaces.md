# 5.4 Interfaces de Almacenamiento

## Definición

La interfaz de almacenamiento es el protocolo y conexión física que comunica el dispositivo de almacenamiento con el sistema. SATA (Serial ATA) es la interfaz legacy con un protocolo AHCI diseñado para HDDs. PCIe (Peripheral Component Interconnect Express) es el bus de alta velocidad que conecta dispositivos directamente al CPU. NVMe (Non-Volatile Memory Express) es el protocolo diseñado específicamente para SSDs sobre PCIe, eliminando el overhead de AHCI.

## Mecánica interna

### SATA

```
  SATA III (SATA 6Gbps):
  - Bandwidth máximo: 6 Gbps = 600 MB/s (teórico)
  - Bandwidth real: ~550 MB/s (overhead de protocolo)
  - Protocolo: AHCI (Advanced Host Controller Interface)
  - Cola de comandos: 32 (NCQ - Native Command Queuing)
  - Conexión: cable punto a punto (1 dispositivo por puerto)

  AHCI fue diseñado para HDDs:
  - 1 cola de comandos de 32 entradas
  - Cada comando requiere interaction con registros del controlador
  - Cada operación genera una interrupción
  - No diseñado para la latencia de submicrosegundos del Flash

  Para un SSD capaz de 500K IOPS:
  AHCI overhead: ~6μs por comando (register writes + interrupt)
  Max IOPS via AHCI: ~160K (limitado por el protocolo, no el SSD)
  → AHCI es el bottleneck.
```

### PCIe

```
  PCIe es un bus serial de alta velocidad, punto a punto.

  PCIe lanes y versiones:
  ┌──────────┬────────────────┬──────────────────────────┐
  │ Versión  │ BW por lane    │ ×4 (NVMe típico)         │
  ├──────────┼────────────────┼──────────────────────────┤
  │ PCIe 3.0 │ ~1 GB/s        │ ~3.5 GB/s                │
  │ PCIe 4.0 │ ~2 GB/s        │ ~7 GB/s                  │
  │ PCIe 5.0 │ ~4 GB/s        │ ~14 GB/s                 │
  │ PCIe 6.0 │ ~8 GB/s        │ ~28 GB/s                 │
  └──────────┴────────────────┴──────────────────────────┘

  Cada generación duplica el bandwidth.
  Un NVMe SSD típico usa ×4 lanes.

  PCIe topology:
  CPU ─── PCIe Root Complex
           ├── ×16 GPU
           ├── ×4  NVMe SSD
           ├── ×4  NIC 25Gbps
           └── ×1  Sound card

  El CPU tiene un número limitado de PCIe lanes (~20-128).
  Más dispositivos = las lanes se comparten/reducen.
```

### NVMe Protocol

```
  NVMe fue diseñado desde cero para Flash storage sobre PCIe.

  AHCI vs NVMe:
  ┌────────────────────────┬──────────┬──────────┐
  │                        │ AHCI     │ NVMe     │
  ├────────────────────────┼──────────┼──────────┤
  │ Colas                  │ 1        │ 65,535   │
  │ Depth por cola         │ 32       │ 65,536   │
  │ Submisión de comando   │ Register │ Memory   │
  │                        │ write    │ mapped   │
  │ Interrupciones         │ 1 por    │ MSI-X    │
  │                        │ comando  │ (coalesc)│
  │ Overhead por comando   │ ~6μs     │ ~2.5μs   │
  │ Max IOPS               │ ~160K    │ ~1M+     │
  └────────────────────────┴──────────┴──────────┘

  NVMe Submission/Completion Queue:
  ┌─────────────────────────────────────────────────┐
  │ CPU Core 0: SQ[0] ──▶ NVMe Controller ──▶ CQ[0]│
  │ CPU Core 1: SQ[1] ──▶ NVMe Controller ──▶ CQ[1]│
  │ ...                                             │
  │ CPU Core N: SQ[N] ──▶ NVMe Controller ──▶ CQ[N]│
  └─────────────────────────────────────────────────┘

  Cada core tiene su PROPIO par de colas.
  No hay contención entre cores (no se necesita lock).
  El controller procesa comandos de todas las colas en paralelo.

  Submission Queue (SQ): el host escribe comandos (read, write, etc.)
  Completion Queue (CQ): el controller escribe resultados
  Doorbell registers: notifican al controller de nuevos comandos

  io_uring (Linux): interfaz de kernel que explota el modelo
  de submission/completion de NVMe → mínimo overhead por I/O.
```

## Modelo mental

SATA/AHCI es una carretera de un carril: los coches (comandos) pasan de uno en uno con semáforo (interrupciones) en cada cruce. PCIe/NVMe es una autopista de múltiples carriles con peaje electrónico: miles de coches circulan en paralelo sin detenerse. El cuello de botella de un SSD SATA no es el SSD sino la interfaz. NVMe elimina ese cuello de botella.

## Insights de ingeniería

- **SATA SSDs son un compromiso obsoleto**: si vas a comprar un SSD, que sea NVMe. La diferencia de costo es mínima (~10-20%) pero la diferencia de rendimiento es 5-10× en IOPS y 3-7× en bandwidth secuencial.
- **io_uring + NVMe es la combinación óptima para I/O**: io_uring evita syscalls por operación (batch submission), y NVMe evita locks entre cores (per-core queues). Juntos, pueden alcanzar millones de IOPS con overhead mínimo de CPU.
