# 6.1 Tipos de Buses

## Definición

Un bus es un sistema de comunicación que transfiere datos entre componentes de una computadora. Los tres tipos fundamentales son: el bus de datos (transporta los datos), el bus de direcciones (indica la ubicación de memoria o dispositivo destino), y el bus de control (transporta señales de sincronización y control como read/write, interrupt, clock).

## Mecánica interna

```
  Arquitectura de bus clásica:

  ┌─────┐     ┌─────────────────────────────────┐
  │ CPU │◀───▶│         System Bus               │
  └─────┘     │  ┌──────────────────────────┐    │
              │  │ Address Bus (64 bits)     │    │
              │  │ → CPU envía dirección     │    │
              │  ├──────────────────────────┤    │
              │  │ Data Bus (64 bits)        │    │
              │  │ ← → datos bidireccional  │    │
              │  ├──────────────────────────┤    │
              │  │ Control Bus               │    │
              │  │ → R/W, IRQ, clock, etc.  │    │
              │  └──────────────────────────┘    │
              └──────┬──────────┬───────────┘
                     │          │
                  ┌──┴──┐   ┌──┴──┐
                  │ RAM │   │ I/O │
                  └─────┘   └─────┘


  Bus de datos:
  Transporta instrucciones y datos entre CPU, memoria, y dispositivos.
  Ancho: 64 bits en arquitecturas modernas (8 bytes por transferencia).
  Bidireccional: la CPU puede leer y escribir.

  Bus de direcciones:
  Transporta la dirección de memoria o puerto I/O objetivo.
  Ancho: 48-52 bits en x86-64 (256TB de espacio direccionable).
  Unidireccional: la CPU genera la dirección.
  2^48 bits = 256 TB de espacio de direcciones virtuales.

  Bus de control:
  Señales de sincronización y comando:
  - Read/Write: dirección de la operación
  - Memory/IO: indica si el destino es memoria o dispositivo I/O
  - IRQ (Interrupt Request): dispositivo solicita atención del CPU
  - Bus Request/Grant: arbitraje de acceso al bus compartido
  - Clock: señal de sincronización


  Bus compartido vs punto a punto:

  Compartido (legacy): múltiples dispositivos comparten el mismo bus.
  Solo uno puede transmitir a la vez → requiere arbitraje.
  Ejemplo: PCI, ISA.

  Punto a punto (moderno): cada dispositivo tiene su propio enlace.
  Múltiples dispositivos transmiten simultáneamente.
  Ejemplo: PCIe, QPI/UPI (inter-CPU), HyperTransport.
```

## Modelo mental

Los buses son las carreteras del sistema. El bus de datos es la carga que transportan los camiones. El bus de direcciones es el GPS que indica el destino. El bus de control son las señales de tránsito (semáforos, señales de dirección). Los buses compartidos son una carretera de un carril (un vehículo a la vez); los punto a punto son autopistas con carriles dedicados.

## Insights de ingeniería

- **Los buses compartidos son historia**: en sistemas modernos, todo es punto a punto (PCIe lanes, UPI links). La razón es que un bus compartido se satura rápidamente con múltiples dispositivos de alta velocidad. PCIe resuelve esto dando a cada dispositivo su propio enlace dedicado.
- **El ancho del bus de datos determina el throughput por ciclo**: un bus de 64 bits transfiere 8 bytes por ciclo. A 4GHz, eso es 32 GB/s teóricos. En la práctica, overhead de protocolo y latencia de arbitraje reducen el throughput real.
