# 6.3 DMA (Direct Memory Access)

## Definición

DMA es un mecanismo que permite a dispositivos periféricos (NIC, SSD, GPU) transferir datos directamente a/desde la memoria principal sin involucrar a la CPU en cada byte transferido. Sin DMA, la CPU tendría que ejecutar instrucciones de lectura/escritura para cada word de datos (programmed I/O), consumiendo ciclos de CPU que podrían usarse para cómputo.

## Mecánica interna

```
  Sin DMA (Programmed I/O):

  Para transferir 4KB de disco a memoria:
  CPU: read word from device → store to memory → repeat × 512 (4KB/8 bytes)
  512 instrucciones de CPU consumidas, CPU no puede hacer nada más.


  Con DMA:

  1. CPU programa el DMA controller:
     - Dirección fuente (dispositivo o memoria)
     - Dirección destino (memoria o dispositivo)
     - Tamaño de transferencia
     - Dirección (device→memory o memory→device)

  2. DMA controller ejecuta la transferencia:
     El dispositivo escribe directamente a RAM a través del bus,
     sin que cada byte pase por la CPU.

  3. Al terminar, DMA controller envía una interrupción al CPU:
     "Transferencia completa. Los datos están en memoria."

  4. CPU puede procesar los datos.


  ┌─────┐  1. Setup    ┌──────────┐  2. Transfer  ┌─────┐
  │ CPU │────────────▶│ DMA      │───────────────▶│ RAM │
  │     │◀────────────│Controller│◀───────────────│     │
  │     │  4. IRQ     │          │                │     │
  └─────┘             └────┬─────┘                └─────┘
                           │ 3. Data
                      ┌────┴─────┐
                      │ Device   │
                      │ (NIC/SSD)│
                      └──────────┘


  Scatter-Gather DMA:
  En vez de una transferencia contigua, el DMA controller puede
  transferir datos a/desde MÚLTIPLES regiones de memoria no contiguas
  en una sola operación.

  Scatter (device → memoria):
  Datos del dispositivo → [buffer 1][buffer 2][buffer 3]

  Gather (memoria → device):
  [header][payload][trailer] → datos al dispositivo

  Esto es esencial para networking:
  Un paquete de red se construye combinando buffers separados
  (headers en un buffer, datos en otro) sin necesidad de copiarlos
  a un buffer contiguo.
```

### DMA en dispositivos modernos

```
  NIC con DMA:
  1. Paquete llega a la NIC
  2. NIC usa DMA para escribir el paquete directamente en un
     buffer de memoria (ring buffer del kernel)
  3. NIC envía interrupción al CPU
  4. CPU procesa el paquete desde el buffer en memoria

  NVMe con DMA:
  1. CPU escribe comando de lectura en Submission Queue (en memoria)
  2. CPU escribe doorbell register para notificar al controlador
  3. Controlador NVMe lee la SQ via DMA
  4. Controlador lee datos del NAND
  5. Controlador escribe datos a memoria via DMA
  6. Controlador escribe Completion Queue entry via DMA
  7. Controlador envía interrupción MSI-X al CPU

  GPU con DMA:
  1. CPU sube texturas/datos a la GPU via DMA (PCIe transfer)
  2. GPU procesa
  3. GPU escribe resultado a memoria del host via DMA

  IOMMU (Input/Output MMU):
  Traduce direcciones virtuales de dispositivos a físicas.
  Permite que dispositivos usen direcciones virtuales (no físicas)
  → seguridad (dispositivo no puede acceder a memoria arbitraria)
  → facilita virtualización (VMs pueden dar acceso directo a dispositivos)
```

## Modelo mental

DMA es un servicio de mensajería que mueve paquetes entre el almacén (dispositivo) y tu oficina (RAM) sin que tú tengas que ir personalmente a buscarlos. Tú (CPU) solo dices "trae esa caja de la dirección A al escritorio B" y luego sigues trabajando. Cuando el mensajero termina, toca tu puerta (interrupción) y dices "gracias" y procesas el paquete.

## Modos de fallo

| Síntoma | Causa | Solución |
|---------|-------|----------|
| CPU al 100% durante I/O | DMA deshabilitado, usando PIO | Verificar que DMA esté habilitado |
| Cache incoherencia post-DMA | DMA escribe a RAM, CPU cache tiene dato viejo | Cache invalidation antes de leer datos DMA |
| Seguridad: dispositivo malicioso accede RAM | Sin IOMMU | Habilitar IOMMU (Intel VT-d, AMD-Vi) |

## Insights de ingeniería

- **Sin DMA, las NICs de alta velocidad serían imposibles**: una NIC de 100 Gbps recibe ~8.3 millones de paquetes/segundo de 1500 bytes. Con PIO, la CPU necesitaría ~1.5 billones de instrucciones/segundo solo para copiar datos. Con DMA, la CPU solo procesa interrupciones y paquetes ya en memoria.
- **Cache coherencia con DMA requiere cuidado**: cuando un dispositivo escribe datos a RAM via DMA, la CPU cache puede tener la versión vieja. El software debe invalidar las cache lines relevantes antes de leer datos DMA. En architecturas con cache coherent DMA (x86), el hardware maneja esto automáticamente.
- **IOMMU es obligatorio para virtualización segura**: sin IOMMU, un dispositivo con DMA podría leer/escribir cualquier dirección de memoria física, incluyendo memoria de otras VMs o del hypervisor. IOMMU traduce y restringe los accesos DMA, haciendo safe el passthrough de dispositivos a VMs.
