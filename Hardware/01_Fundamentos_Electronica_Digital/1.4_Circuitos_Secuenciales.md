# 1.4 Circuitos Secuenciales

## Definición

Un circuito secuencial es un circuito cuya salida depende no solo de las entradas actuales sino también del estado anterior. A diferencia de los circuitos combinacionales (que son funciones puras), los circuitos secuenciales tienen memoria: almacenan estado. Los flip-flops, latches, registros, y contadores son los bloques que permiten a una computadora recordar datos, mantener un program counter, y sincronizar operaciones con un reloj.

## Mecánica interna

### Latches — almacenamiento sensible a nivel

```
  SR Latch (Set-Reset):
  Construido con dos puertas NOR (o NAND) realimentadas.

  S ──┤NOR├──┬── Q
       │    │
  R ──┤NOR├──┴── Q'

  ┌───┬───┬─────┬────────────────────────────┐
  │ S │ R │  Q  │ Comportamiento              │
  ├───┼───┼─────┼────────────────────────────┤
  │ 0 │ 0 │ Q_prev │ Mantiene estado anterior │
  │ 1 │ 0 │  1  │ SET: Q se pone en 1        │
  │ 0 │ 1 │  0  │ RESET: Q se pone en 0      │
  │ 1 │ 1 │  ?  │ PROHIBIDO: estado inválido  │
  └───┴───┴─────┴────────────────────────────┘

  El SR latch es transparente: cuando S o R cambian, Q cambia
  inmediatamente (no espera al reloj). Esto es problemático en
  circuitos síncronos porque los datos pueden cambiar en cualquier momento.


  D Latch (Data Latch):
  Soluciona el estado prohibido del SR: tiene una sola entrada de datos (D).

  ┌───┬────┬─────┬────────────────────────────┐
  │ EN│ D  │  Q  │ Comportamiento              │
  ├───┼────┼─────┼────────────────────────────┤
  │ 0 │ X  │ Q_prev │ Mantiene estado          │
  │ 1 │ 0  │  0  │ Q sigue a D                 │
  │ 1 │ 1  │  1  │ Q sigue a D                 │
  └───┴────┴─────┴────────────────────────────┘

  Cuando EN (enable) = 1: Q = D (transparente, la salida sigue a la entrada)
  Cuando EN = 0: Q mantiene el último valor capturado (almacenamiento)

  Problema: durante el tiempo que EN = 1, cualquier cambio en D
  se propaga directamente a Q. Esto puede causar problemas de timing.
```

### Flip-Flops — almacenamiento sensible a flanco

```
  A diferencia del latch, un flip-flop captura datos SOLO en el
  instante del flanco del reloj (rising edge o falling edge).
  El resto del tiempo, la salida es estable.

  D Flip-Flop (el más usado en diseño digital):

  CLK ──┐
  D  ───┤ D-FF ├── Q
        └──────┘

  Comportamiento:
  - En el rising edge de CLK: Q captura el valor actual de D
  - En cualquier otro momento: Q mantiene su valor
  - D puede cambiar libremente entre flancos sin afectar Q

  Esto resuelve el problema del latch:
  Los datos se capturan en un instante discreto, no durante un intervalo.


  Timing constraints:
  ┌──────────────────────────────────────────────────────┐
  │                                                      │
  │  Setup time (tsu): D debe estar estable ANTES del    │
  │  flanco del reloj por al menos tsu.                  │
  │                                                      │
  │  Hold time (th): D debe permanecer estable DESPUÉS   │
  │  del flanco del reloj por al menos th.               │
  │                                                      │
  │  ──── D estable ────┤ CLK edge ├── D estable ───     │
  │  ←────── tsu ───────┤          ├──── th ────→        │
  │                                                      │
  │  Si D cambia dentro de la ventana tsu + th:          │
  │  → Metaestabilidad: Q queda en un estado intermedio  │
  │    (ni 0 ni 1) por un tiempo indeterminado.          │
  │  → El circuito puede producir resultados incorrectos.│
  │                                                      │
  │  Clock-to-Q delay (tCQ): tiempo desde el flanco      │
  │  hasta que Q tiene un valor estable y válido.        │
  └──────────────────────────────────────────────────────┘


  JK Flip-Flop:
  Similar al SR pero sin estado prohibido.
  ┌───┬───┬─────────────────────┐
  │ J │ K │ Q (next)            │
  ├───┼───┼─────────────────────┤
  │ 0 │ 0 │ Q (mantiene)        │
  │ 0 │ 1 │ 0 (reset)           │
  │ 1 │ 0 │ 1 (set)             │
  │ 1 │ 1 │ Q' (toggle)         │
  └───┴───┴─────────────────────┘

  El modo toggle (J=K=1) es útil para construir contadores.


  T Flip-Flop (Toggle):
  Caso especial del JK con J=K=T.
  Si T=1: Q se invierte en cada flanco de reloj.
  Si T=0: Q mantiene su valor.
  Base de los contadores binarios.
```

### Registros

```
  Un registro es un grupo de flip-flops que almacenan múltiples bits
  en paralelo, todos controlados por el mismo reloj.

  Registro de 8 bits:
  ┌─────────────────────────────────────────┐
  │ D7 D6 D5 D4 D3 D2 D1 D0   ← entradas  │
  │ ┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐      │
  │ │FF││FF││FF││FF││FF││FF││FF││FF│ ← CLK │
  │ └──┘└──┘└──┘└──┘└──┘└──┘└──┘└──┘      │
  │ Q7 Q6 Q5 Q4 Q3 Q2 Q1 Q0   ← salidas   │
  └─────────────────────────────────────────┘

  Tipos en una CPU:
  - Program Counter (PC): almacena la dirección de la próxima instrucción
  - Instruction Register (IR): almacena la instrucción actual
  - General Purpose Registers (GPR): almacenan datos de trabajo
    x86-64: RAX, RBX, RCX, RDX, RSI, RDI, R8-R15 (16 registros de 64 bits)
    ARM64: X0-X30 (31 registros de 64 bits)
  - Stack Pointer (SP): apunta al tope del stack
  - Flags/Status Register: carry, zero, overflow, sign

  Acceso a registros: ~0.3-0.5ns (dentro del core, sin latencia de bus)
  Acceso a L1 cache: ~1-2ns
  Acceso a RAM: ~50-100ns

  Los registros son la memoria más rápida que existe: están físicamente
  dentro de la ALU, conectados por cables directos, sin pasar por bus ni cache.


  Shift Register:
  Los bits se desplazan una posición por cada ciclo de reloj.

  Ciclo 0: [1][0][1][1]
  Ciclo 1: [X][1][0][1]  ← shift right, bit más significativo se pierde o recircula

  Usado en: comunicación serial (UART convierte paralelo a serial con shift register),
  generación de secuencias pseudo-aleatorias (LFSR).
```

### Contadores

```
  Un contador es una cadena de flip-flops que incrementa (o decrementa)
  su valor binario en cada flanco de reloj.

  Contador binario de 4 bits (asíncrono / ripple counter):

  CLK ──▶ FF0 (Q0 = bit 0)
           Q0 ──▶ FF1 (Q1 = bit 1)
                   Q1 ──▶ FF2 (Q2 = bit 2)
                           Q2 ──▶ FF3 (Q3 = bit 3)

  Cada flip-flop divide la frecuencia por 2:
  CLK: ┌┐┌┐┌┐┌┐┌┐┌┐┌┐┌┐
  Q0:  ┌──┐┌──┐┌──┐┌──┐
  Q1:  ┌────┐┌────┐┌────┐
  Q2:  ┌────────┐┌────────┐
  Q3:  ┌────────────────┐

  Secuencia: 0000 → 0001 → 0010 → ... → 1111 → 0000 (overflow)

  Ripple counter: cada FF dispara al siguiente → delay acumulado.
  Synchronous counter: todos los FF comparten el mismo CLK → más rápido pero más gates.

  Usos en computación:
  - Timer/counter hardware en microcontroladores
  - Refresh counter en DRAM (cada fila debe refrescarse periódicamente)
  - Performance counters en CPUs (cuentan instrucciones, cache misses, etc.)
  - Watchdog timer: si el contador llega al máximo sin reset → fallo detectado
```

## Modelo mental

Los circuitos combinacionales son funciones puras: misma entrada, misma salida, sin memoria. Los circuitos secuenciales son funciones con estado: su salida depende de la historia. Un flip-flop es el átomo de memoria: almacena un bit. Un registro es una palabra de flip-flops. Un contador es una máquina que avanza por una secuencia de estados. Con flip-flops puedes recordar; con contadores puedes contar; con registros puedes almacenar instrucciones y datos. La CPU entera es una máquina de estados finita construida sobre estos bloques: en cada flanco de reloj, los registros capturan nuevos datos, el program counter avanza, y la ALU produce un resultado.

## Modos de fallo

| Síntoma | Causa | Solución |
|---------|-------|----------|
| Datos corruptos entre clock domains | Metaestabilidad al cruzar dominios | Sincronizadores de doble flip-flop |
| Glitch en salida de contador | Ripple delay en contador asíncrono | Usar contador síncrono |
| Timing violation en pipeline | Setup time violado (path demasiado largo) | Reducir lógica combinacional entre stages |
| Register file corruption | Write-after-write hazard sin forwarding | Forwarding logic en pipeline |

## Insights de ingeniería

- **El D flip-flop es el bloque más importante de toda la computación digital**: cada registro de CPU, cada celda de SRAM (cache), cada etapa de pipeline, cada buffer — todos se construyen con D flip-flops. Entender su timing (setup, hold, clock-to-Q) es entender los límites fundamentales de velocidad del hardware.
- **Metaestabilidad es un problema real en sistemas multi-clock**: cuando un dato cruza de un dominio de reloj a otro (e.g., dato de la NIC entrando al dominio del CPU), puede violar setup/hold time y causar metaestabilidad. La solución estándar es un sincronizador de 2-3 flip-flops en serie, que reduce la probabilidad de metaestabilidad a niveles insignificantes a costa de 2-3 ciclos de latencia.
- **Los contadores de rendimiento del CPU son contadores hardware reales**: cuando usas `perf stat` en Linux para medir cache misses o branch mispredictions, estás leyendo contadores hardware — circuitos contadores físicos dentro del CPU que se incrementan cada vez que ocurre un evento. Son zero-overhead porque no involucran software.
