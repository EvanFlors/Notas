# 1.1 Magnitudes Eléctricas

## Definición

Las magnitudes eléctricas son las propiedades físicas fundamentales que gobiernan el comportamiento de los circuitos electrónicos. Toda computadora, desde un microcontrolador hasta un servidor de datacenter, opera manipulando voltaje y corriente en circuitos integrados. Comprender estas magnitudes es entender el sustrato físico sobre el que corre todo software.

## Mecánica interna

### Voltaje (V) — Diferencia de potencial

```
  El voltaje es la "presión" que empuja a los electrones a través de un circuito.
  Se mide en Voltios (V).

  Analogía hidráulica:
  ┌─────────────────────────────────┐
  │ Tanque alto ──────▶ Tanque bajo │
  │ (mayor potencial)  (menor pot.) │
  │                                 │
  │ La diferencia de altura = voltaje│
  │ El agua fluye de alto a bajo    │
  └─────────────────────────────────┘

  En un circuito digital:
  - Nivel lógico HIGH: típicamente 3.3V o 5V (dependiendo de la tecnología)
  - Nivel lógico LOW: típicamente 0V (GND)
  - El procesador discrimina entre HIGH y LOW con umbrales:
    V > 2.0V → HIGH (lógico 1)
    V < 0.8V → LOW  (lógico 0)
    Entre 0.8V y 2.0V → zona indefinida (no permitido en estado estable)

  Los procesadores modernos operan a voltajes muy bajos (~0.7V-1.2V)
  para reducir el consumo de energía y la generación de calor.
```

### Corriente (I) — Flujo de carga

```
  La corriente es la cantidad de carga eléctrica que fluye por un conductor
  por unidad de tiempo. Se mide en Amperios (A).

  I = Q / t
  donde Q = carga en Coulombs, t = tiempo en segundos

  1 Amperio = 1 Coulomb/segundo ≈ 6.24 × 10^18 electrones por segundo

  En circuitos digitales:
  - La corriente fluye a través de los transistores cuando conmutan.
  - Más transistores conmutando = más corriente = más calor.
  - Un CPU moderno puede consumir 100-200A a 1V bajo carga máxima.

  Corriente de fuga (leakage current):
  Incluso cuando un transistor está "apagado", una pequeña corriente fluye.
  A escalas nanométricas (5nm, 3nm), la fuga es un problema significativo:
  contribuye al consumo de energía incluso en idle.
```

### Resistencia (R) — Oposición al flujo

```
  La resistencia es la oposición que un material presenta al flujo de corriente.
  Se mide en Ohmios (Ω).

  Conductores: baja resistencia (cobre, aluminio) → usados en cables y trazas de PCB
  Aislantes: alta resistencia (vidrio, plástico) → usados para separar conductores
  Semiconductores: resistencia variable (silicio) → base de los transistores

  En un transistor MOSFET (la base de toda CPU moderna):
  - Gate voltage controla la resistencia del canal
  - V_gate HIGH → canal conductor (resistencia baja) → corriente fluye → ON
  - V_gate LOW  → canal aislante (resistencia alta) → corriente no fluye → OFF
  - Este comportamiento ON/OFF es la base de la lógica digital: 1 y 0
```

### Ley de Ohm

```
  V = I × R

  Voltaje (V) = Corriente (A) × Resistencia (Ω)

  Relaciones derivadas:
  I = V / R  → a más voltaje, más corriente (con resistencia constante)
  R = V / I  → a más corriente para el mismo voltaje, menor resistencia

  Ejemplo práctico:
  Un LED con resistencia de 330Ω conectado a 3.3V:
  I = 3.3V / 330Ω = 0.01A = 10mA ← corriente que circula por el LED

  En un procesador:
  Si el CPU opera a 1.0V y consume 150A:
  R_efectiva = 1.0V / 150A = 0.0067Ω ← resistencia extremadamente baja
  Esto es por qué los VRM (Voltage Regulator Modules) deben entregar
  corrientes enormes con precisión.
```

### Potencia (P) — Energía por unidad de tiempo

```
  P = V × I         (Watts)
  P = I² × R        (forma equivalente)
  P = V² / R        (forma equivalente)

  La potencia determina cuánta energía se disipa como calor.

  Potencia en un CPU:
  ┌──────────────────────────────────────────────┐
  │ P_dinámica = C × V² × f                     │
  │   C = capacitancia (proporcional al # de transistores)
  │   V = voltaje operativo                      │
  │   f = frecuencia de reloj                    │
  │                                              │
  │ P_estática = V × I_fuga                      │
  │   I_fuga = corriente de fuga (leakage)       │
  │                                              │
  │ P_total = P_dinámica + P_estática            │
  └──────────────────────────────────────────────┘

  Observaciones clave:
  - P_dinámica escala con V²: reducir voltaje de 1.2V a 1.0V reduce
    la potencia dinámica en un 30% (0.83² vs 1.44²).
  - P_dinámica escala linealmente con frecuencia: duplicar frecuencia
    duplica la potencia (y el calor).
  - Esto explica por qué los CPUs no siguen subiendo de frecuencia:
    a 5GHz+ el calor generado es inmanejable con refrigeración por aire.

  TDP (Thermal Design Power):
  La potencia máxima que el sistema de refrigeración debe disipar.
  Un CPU con TDP de 125W necesita un cooler capaz de extraer 125W de calor.

  Ejemplo: Intel i9-13900K
  - TDP base: 125W
  - PBP (Processor Base Power): 125W
  - MTP (Maximum Turbo Power): 253W ← bajo carga máxima, consume el doble
```

### Relación entre magnitudes

```
  ┌──────────────────────────────────────────────────────────┐
  │                                                          │
  │  V = I × R          ← Ley de Ohm                       │
  │  P = V × I          ← Potencia                         │
  │  P = V² / R         ← Potencia en función de V y R     │
  │  P = I² × R         ← Potencia en función de I y R     │
  │  E = P × t          ← Energía = potencia × tiempo      │
  │                                                          │
  │  En un CPU:                                              │
  │  P_dinámica = C × V² × f  ← la ecuación clave          │
  │                                                          │
  └──────────────────────────────────────────────────────────┘
```

## Modelo mental

Un circuito eléctrico es un sistema hidráulico: el voltaje es la presión del agua, la corriente es el caudal, la resistencia es el estrechamiento del tubo, y la potencia es la fuerza con la que el agua golpea la turbina. En una computadora, los transistores son válvulas controladas por voltaje: abren y cierran el flujo de corriente billones de veces por segundo, y cada apertura/cierre consume energía y genera calor. La ecuación P = C × V² × f gobierna todo: explica por qué se baja el voltaje (eficiencia), por qué hay un límite práctico de frecuencia (calor), y por qué los chips más pequeños (menor C) son más eficientes.

## Modos de fallo

| Síntoma | Causa | Solución |
|---------|-------|----------|
| CPU thermal throttling | P_total excede capacidad del cooler | Mejorar refrigeración o reducir frecuencia/voltaje |
| Inestabilidad del sistema | Voltaje insuficiente para la frecuencia | Aumentar voltaje (más calor) o reducir frecuencia |
| Consumo excesivo en idle | Corriente de fuga alta (proceso nanométrico) | DVFS (Dynamic Voltage and Frequency Scaling) |
| VRM sobrecalentado | Corriente excesiva (>150A) | VRM de mayor capacidad, mejor airflow |

## Herramientas de diagnóstico

```
  Linux:
  - turbostat: muestra frecuencia, voltaje, potencia, y temperatura por core
  - sensors (lm-sensors): lecturas de sensores de temperatura y voltaje
  - powertop: análisis de consumo de energía
  - /sys/class/hwmon/: interfaz directa a sensores de hardware

  Métricas relevantes:
  - Package power (W): consumo total del procesador
  - Core voltage (V): voltaje operativo actual
  - Core temperature (°C): temperatura por core
  - Frequency (GHz): frecuencia actual por core
```

## Insights de ingeniería

- **DVFS es el mecanismo clave de eficiencia energética**: Dynamic Voltage and Frequency Scaling permite al CPU reducir voltaje y frecuencia cuando la carga es baja. Dado que P = C × V² × f, reducir ambos simultáneamente tiene un efecto cúbico en la potencia. Un servidor en idle puede consumir 30% de su potencia máxima gracias a DVFS.
- **El voltaje es el enemigo de la escala**: cada generación de procesadores intenta reducir el voltaje operativo porque la potencia escala con V². Pasar de 14nm a 7nm permite operar a menor voltaje con la misma frecuencia, reduciendo potencia y calor.
- **Power delivery es un problema de ingeniería real**: entregar 150A a 1V de forma estable requiere VRMs (Voltage Regulator Modules) de alta calidad y un PCB con trazas de baja impedancia. En servidores de datacenter, la infraestructura de potencia (PSU, distribución, UPS) es tan crítica como el hardware de cómputo.
