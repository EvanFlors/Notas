# 1.2 Señales

## Definición

Una señal es una magnitud física que varía en el tiempo y transporta información. En electrónica, las señales son variaciones de voltaje o corriente. Los sistemas digitales operan con señales discretas (dos niveles: HIGH y LOW), mientras que los sistemas analógicos operan con señales continuas. La conversión entre ambos mundos (ADC/DAC) y la comprensión de ruido, frecuencia, y timing son fundamentales para entender cómo el hardware procesa datos.

## Mecánica interna

### Señales analógicas vs digitales

```
  Señal analógica:
  Voltaje varía continuamente en el tiempo. Puede tomar cualquier valor.

  V │    ╱╲
    │   ╱  ╲      ╱╲
    │  ╱    ╲    ╱  ╲
    │ ╱      ╲  ╱    ╲
    │╱        ╲╱      ╲
    └─────────────────── t

  Ejemplos: señal de audio de un micrófono, lectura de sensor de temperatura.
  Información codificada en la amplitud, frecuencia, o fase de la onda.


  Señal digital:
  Voltaje toma solo dos valores discretos: HIGH (1) y LOW (0).

  V │ ┌──┐  ┌──┐     ┌──┐
    │ │  │  │  │     │  │
    │─┘  └──┘  └─────┘  └──
    │
    └─────────────────────── t

  Ejemplos: datos en un bus de CPU, señales de reloj, comunicación serial.
  Información codificada en la secuencia de 1s y 0s.


  Conversión:
  ADC (Analog-to-Digital Converter):
    Señal analógica → muestreo a intervalos regulares → cuantización → bits
    Resolución: 8-bit ADC = 256 niveles, 16-bit = 65536 niveles
    Teorema de Nyquist: frecuencia de muestreo ≥ 2× frecuencia máxima de la señal

  DAC (Digital-to-Analog Converter):
    Secuencia de bits → voltaje analógico proporcional
    Usado en: salida de audio, control de motores
```

### Ruido y tolerancia

```
  El ruido es cualquier perturbación no deseada que se superpone a la señal.

  Señal ideal:        Señal con ruido:
  ┌──┐  ┌──┐         ┌╌╌┐  ┌╌╌┐
  │  │  │  │         ╎  ╎  ╎  ╎   ← fluctuaciones
  ┘  └──┘  └         ┘  └╌╌┘  └

  Fuentes de ruido:
  - Crosstalk: señal en una traza induce voltaje en trazas adyacentes
  - EMI (Electromagnetic Interference): interferencia de fuentes externas
  - Ruido térmico (Johnson-Nyquist): fluctuaciones aleatorias por temperatura
  - Ruido de conmutación (switching noise): transitorio al cambiar de estado

  Margen de ruido (noise margin):
  ┌────────────────────────────────────────────┐
  │ V_OH (Output High) = 2.4V                 │
  │ V_IH (Input High)  = 2.0V                 │
  │ Margen HIGH = V_OH - V_IH = 0.4V          │
  │                                            │
  │ V_OL (Output Low)  = 0.4V                 │
  │ V_IL (Input Low)   = 0.8V                 │
  │ Margen LOW = V_IL - V_OL = 0.4V           │
  │                                            │
  │ Si el ruido < 0.4V, la señal se interpreta │
  │ correctamente. Si el ruido > 0.4V,         │
  │ puede haber un bit flip (error).           │
  └────────────────────────────────────────────┘

  Señal diferencial:
  En vez de medir voltaje respecto a GND, se miden dos conductores
  donde la señal es la DIFERENCIA entre ellos.
  El ruido afecta ambos conductores por igual → se cancela al restar.
  Usado en: PCIe, USB, HDMI, Ethernet (todas las interfaces modernas de alta velocidad).
```

### Frecuencia, periodo y duty cycle

```
  Frecuencia (f): número de ciclos por segundo. Unidad: Hertz (Hz).
  Periodo (T): duración de un ciclo. T = 1/f.

  Reloj de un CPU a 4 GHz:
  f = 4 × 10^9 Hz
  T = 1 / (4 × 10^9) = 0.25 ns = 250 picosegundos por ciclo

  ┌──┐  ┌──┐  ┌──┐
  │  │  │  │  │  │
  ──┘  └──┘  └──┘  └──
  ├──T──┤

  En 250ps, la luz viaja ~7.5cm.
  Esto impone un límite físico: las señales no pueden recorrer
  distancias largas en un solo ciclo de reloj a frecuencias altas.


  Duty cycle: porcentaje del periodo que la señal está en HIGH.

  50% duty cycle (reloj estándar):
  ┌──┐  ┌──┐
  │  │  │  │
  ──┘  └──┘  └──
  HIGH = LOW = T/2

  75% duty cycle:
  ┌────┐┌────┐
  │    ││    │
  ──┘  └┘    └
  HIGH = 3T/4, LOW = T/4

  El duty cycle importa en:
  - PWM (Pulse Width Modulation): control de velocidad de ventiladores,
    brillo de LEDs, control de motores. El duty cycle promedia
    el voltaje efectivo: 50% duty en 5V = 2.5V efectivos.
  - Señales de reloj: un duty cycle de 50% es ideal para sincronización.
```

### Señal de reloj (clock signal)

```
  El reloj es la señal maestra que sincroniza todas las operaciones
  de un circuito digital. Cada transición (rising edge o falling edge)
  marca un instante donde los circuitos pueden capturar o propagar datos.

  Rising edge         Falling edge
      ↓                    ↓
  ┌───┐    ┌───┐    ┌───┐
  │   │    │   │    │   │
  ┘   └────┘   └────┘   └────

  Clock distribution:
  El reloj debe llegar a TODOS los componentes del chip al mismo tiempo.
  A 4GHz, un desfase de 50ps puede causar que un flip-flop capture
  datos incorrectos (timing violation).

  Clock skew: diferencia de tiempo de llegada del reloj entre dos puntos del chip.
  Clock jitter: variación aleatoria en el periodo del reloj entre ciclos consecutivos.

  Ambos son enemigos del rendimiento:
  - Skew excesivo → setup/hold time violations → datos corruptos
  - Jitter excesivo → reduce el margen de timing → límite de frecuencia máxima

  Clock domains:
  Un chip moderno tiene MÚLTIPLES dominios de reloj:
  - CPU cores: 4-5 GHz
  - Uncore/ring bus: 2-3 GHz
  - Memory controller: ~1.6 GHz (DDR5-6400 / 2)
  - PCIe controller: 2.5 GHz (PCIe 4.0 base clock)

  Cruzar dominios de reloj requiere sincronizadores para evitar metaestabilidad.
```

## Modelo mental

Una señal digital es un acuerdo binario: por encima de cierto voltaje es 1, por debajo es 0. El ruido intenta corromper ese acuerdo; los márgenes de ruido y las señales diferenciales lo protegen. El reloj es el latido del corazón del sistema: cada pulso marca un instante discreto donde los circuitos pueden actuar. La frecuencia del reloj determina cuántos de esos instantes hay por segundo, y por tanto la velocidad máxima de procesamiento. Pero la frecuencia tiene un techo: a 4GHz, un ciclo dura 250ps, distancia en la que la luz recorre ~7.5cm. A esa escala, la física limita qué tan rápido y qué tan lejos pueden viajar las señales.

## Modos de fallo

| Síntoma | Causa | Solución |
|---------|-------|----------|
| Bit errors en transmisión | Ruido excede margen de ruido | Señales diferenciales, mejor shielding |
| Sistema inestable al overclockear | Setup/hold time violations por frecuencia excesiva | Reducir frecuencia o aumentar voltaje |
| Datos corruptos entre componentes | Clock skew entre dominios | Sincronizadores de clock domain crossing |
| Errores intermitentes en RAM | Jitter en el reloj de memoria | Timings de RAM más conservadores |

## Insights de ingeniería

- **Las señales diferenciales dominan las interfaces modernas**: PCIe, USB, SATA, Ethernet, HDMI — todas usan señales diferenciales. La razón es simple: el ruido se cancela al restar las dos líneas, permitiendo frecuencias mucho más altas que las señales single-ended. Cuando ves "PCIe Gen 5 a 32 GT/s", eso es posible solo porque las señales son diferenciales.
- **El clock distribution tree es una de las partes más difíciles del diseño de chips**: hacer que una señal de 4GHz llegue simultáneamente a billones de transistores distribuidos en un die de ~250mm² requiere ingeniería de precisión al nivel de picosegundos. El consumo del árbol de distribución del reloj puede ser 20-30% del consumo total del chip.
- **La frecuencia de reloj dejó de ser la métrica principal**: entre 2000 y 2005, la "carrera de GHz" se detuvo porque P = C × V² × f hacía que más frecuencia generase calor insostenible. Desde entonces, el rendimiento se escala con más cores, mejor IPC, y caches más grandes — no con más GHz.
