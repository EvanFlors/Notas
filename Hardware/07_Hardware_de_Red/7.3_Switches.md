# 7.3 Switches

## Definición

Un switch de red opera en capa 2 (enlace de datos) del modelo OSI. Recibe frames Ethernet por un puerto, examina la dirección MAC de destino, y reenvía el frame exclusivamente al puerto donde se encuentra el dispositivo destino. A diferencia de un hub (que reenvía a todos los puertos), un switch segmenta dominios de colisión, permitiendo comunicación simultánea entre múltiples pares de puertos.

## Mecánica interna

### MAC Address Table (CAM Table)

```
  El switch aprende dinámicamente qué MACs están en qué puertos:

  1. Frame llega por puerto 1 con Source MAC AA:BB:CC:DD:EE:01
  2. Switch registra: MAC AA:BB:CC:DD:EE:01 → Puerto 1 (TTL: 300s)
  3. Frame tiene Dest MAC AA:BB:CC:DD:EE:05
  4. Switch busca en CAM table:
     - Si encontrado: reenvía al puerto correspondiente (forwarding)
     - Si NO encontrado: reenvía a TODOS los puertos excepto origen (flooding)

  CAM Table:
  ┌────────────────────┬────────┬─────────┐
  │ MAC Address        │ Puerto │ TTL     │
  ├────────────────────┼────────┼─────────┤
  │ AA:BB:CC:DD:EE:01  │ 1      │ 300s    │
  │ AA:BB:CC:DD:EE:02  │ 2      │ 280s    │
  │ AA:BB:CC:DD:EE:05  │ 5      │ 150s    │
  └────────────────────┴────────┴─────────┘

  CAM table overflow attack:
  Un atacante envía miles de frames con MACs aleatorias.
  CAM table se llena → switch no puede aprender nuevas MACs.
  Frames a MACs desconocidas se envían a TODOS los puertos (flood).
  El atacante puede sniffear tráfico de otros puertos.
  Mitigación: port security (límite de MACs por puerto).
```

### VLANs (802.1Q)

```
  VLAN = red virtual dentro del mismo switch físico.
  Separa dominios de broadcast sin necesidad de switches separados.

  Frame 802.1Q:
  ┌──────┬──────┬────────────┬──────────┬─────────┬─────┐
  │ Dst  │ Src  │ 802.1Q Tag │EtherType │ Payload │ FCS │
  │ MAC  │ MAC  │ (4 bytes)  │          │         │     │
  └──────┴──────┴────────────┴──────────┴─────────┴─────┘
                 │
                 ├─ TPID: 0x8100 (indica que es tagged)
                 ├─ PCP: 3 bits (prioridad, 802.1p QoS)
                 ├─ DEI: 1 bit (drop eligible)
                 └─ VLAN ID: 12 bits (0-4095, 4094 VLANs usables)

  Puerto access: pertenece a 1 VLAN, frames sin tag.
  Puerto trunk: transporta múltiples VLANs, frames con 802.1Q tag.

  Ejemplo:
  VLAN 10: Producción (puertos 1-12)
  VLAN 20: Management (puertos 13-16)
  VLAN 30: Storage (puertos 17-20)
  Trunk: puerto 24 (conecta a otro switch, lleva las 3 VLANs)

  Un frame de VLAN 10 NUNCA llega a dispositivos en VLAN 20
  a menos que un router/L3 switch lo enrute explícitamente.
```

### Spanning Tree Protocol (STP)

```
  Problema: loops en la topología de red.
  Si hay dos caminos entre switch A y switch B,
  un broadcast frame se reenvía infinitamente en loop.

  STP (802.1D) elimina loops deshabilitando puertos redundantes:
  1. Elige un Root Bridge (switch con menor Bridge ID)
  2. Cada switch calcula el camino más corto al Root Bridge
  3. Puertos redundantes se ponen en estado Blocking
  4. Si un enlace falla → STP reconverge (30-50 segundos con STP clásico)

  RSTP (802.1w): convergencia rápida (~1-2 segundos).
  MSTP (802.1s): STP por VLAN o grupo de VLANs.
```

## Modelo mental

Un switch es una oficina de correos inteligente. Cuando llega una carta (frame), mira la dirección del destinatario (MAC destino), consulta su directorio (CAM table), y la envía al buzón correcto (puerto). Si no conoce al destinatario, envía copias a todos los buzones. VLANs son pisos separados del edificio: el correo de un piso nunca llega a otro sin pasar por el ascensor (router).

## Insights de ingeniería

- **L3 switches combinan switching y routing**: en datacenters modernos, los switches ToR (Top of Rack) operan en L3, haciendo routing entre subnets directamente en hardware (ASICs) a velocidad de línea. Esto elimina la necesidad de routers dedicados para tráfico east-west (entre servidores).
- **La latencia de un switch es ~1-5μs (cut-through)**: switches cut-through empiezan a reenviar el frame antes de recibirlo completamente (solo necesitan leer la MAC destino, primeros 14 bytes). Switches store-and-forward esperan el frame completo (para verificar CRC), añadiendo latencia proporcional al tamaño del frame.
