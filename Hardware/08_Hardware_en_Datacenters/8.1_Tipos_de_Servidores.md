# 8.1 Tipos de Servidores

## Definición

Los servidores de datacenter son computadoras diseñadas para operar 24/7 con alta densidad de cómputo, redundancia de componentes, y capacidad de gestión remota. Los formatos principales son rack-mount (1U-4U, montados en racks estándar de 19") y blade (módulos delgados que comparten chasis, fuente de alimentación, y networking).

## Mecánica interna

### Rack-mount Servers

```
  Rack estándar: 42U de altura (1U = 1.75 pulgadas = 44.45mm)

  ┌────────────────────────────────────┐
  │          42U Rack                  │
  │  ┌──────────────────────────────┐  │
  │  │ 1U: ToR Switch (networking) │  │
  │  ├──────────────────────────────┤  │
  │  │ 1U: Server (2×CPU, 512GB)   │  │
  │  ├──────────────────────────────┤  │
  │  │ 1U: Server                  │  │
  │  ├──────────────────────────────┤  │
  │  │ 2U: Storage Server (24×SSD) │  │
  │  ├──────────────────────────────┤  │
  │  │ 4U: GPU Server (8×GPU)      │  │
  │  ├──────────────────────────────┤  │
  │  │         ...                 │  │
  │  ├──────────────────────────────┤  │
  │  │ 1U: PDU (Power Distribution)│  │
  │  └──────────────────────────────┘  │
  └────────────────────────────────────┘

  1U Server (ej: Dell PowerEdge R660):
  - 2× CPU (Intel Xeon o AMD EPYC)
  - 16× DIMM slots (hasta 2TB DDR5)
  - 10× NVMe 2.5" bays
  - 2× PSU redundantes
  - 2× NIC 25GbE onboard
  - BMC (Baseboard Management Controller) con iDRAC/IPMI
  - TDP: 300-600W

  2U Server: más espacio para discos (24 bays), GPUs, o expansión PCIe.
  4U Server: servidores GPU (8× A100/H100, cada GPU consume 300-700W).


  Densidad típica:
  42U rack con 1U servers = 40 servidores (reservando 2U para switches)
  Potencia por rack: 10-30 kW (estándar), hasta 100 kW (GPU-dense)
```

### Blade Servers

```
  Blade Chassis (ej: HPE Synergy, Dell MX7000):

  ┌────────────────────────────────────────┐
  │           Blade Chassis (10U)          │
  │  ┌─────┬─────┬─────┬─────┬─────┐     │
  │  │Blade│Blade│Blade│Blade│Blade│     │
  │  │  1  │  2  │  3  │  4  │  5  │     │
  │  │     │     │     │     │     │ ... │
  │  └─────┴─────┴─────┴─────┴─────┘     │
  │  ┌────────────────────────────────┐    │
  │  │ Shared: PSU, Fans, Networking │    │
  │  │ Midplane (backplane de datos) │    │
  │  └────────────────────────────────┘    │
  └────────────────────────────────────────┘

  Cada blade: CPU + RAM + storage local (mínimo).
  Comparten: fuentes de alimentación, cooling, switches de red.

  Ventajas: mayor densidad, menos cableado, gestión centralizada.
  Desventajas: vendor lock-in, menor flexibilidad de configuración,
  single point of failure en el chassis (midplane, management module).

  Tendencia: los blades han perdido popularidad frente a rack servers
  1U/2U de alta densidad con mejor relación costo/rendimiento.
```

### BMC y gestión remota

```
  BMC (Baseboard Management Controller):
  Procesador ARM independiente en la placa madre del servidor.
  Tiene su propia NIC, OS (Linux embebido), y almacenamiento.
  Funciona incluso con el servidor APAGADO.

  Capacidades:
  - Power on/off/reset remoto
  - Consola serie/KVM virtual (ver la pantalla remotamente)
  - Montar ISO virtual (instalar OS remotamente)
  - Monitoreo de sensores (temperatura, voltaje, fans)
  - Alertas SNMP/email ante eventos críticos
  - Actualización de firmware

  Implementaciones: iDRAC (Dell), iLO (HPE), IPMI (estándar abierto).
  Acceso: interfaz web HTTPS o API REST (Redfish).

  El BMC es esencial: en un datacenter con miles de servidores,
  nadie camina físicamente a un servidor para encenderlo.
```

## Modelo mental

Un rack server es un departamento individual: autosuficiente, con su propia cocina y baño (PSU, cooling). Un blade server es un cuarto de hotel: comparte servicios del edificio (electricidad, agua, recepción) y es más compacto, pero depende de la infraestructura del edificio. El BMC es el portero del edificio que tiene llaves de todos los departamentos y puede abrir/cerrar puertas remotamente.

## Insights de ingeniería

- **La densidad de potencia es el límite real**: un rack de 42U con 40 servidores 1U a 500W cada uno consume 20kW. Un rack de GPU servers (8× H100 por servidor) puede consumir 40-100kW. El datacenter necesita capacidad eléctrica y de refrigeración para esa densidad. Muchos datacenters limitan a 10-15kW por rack, lo que fuerza a distribuir GPU servers en más racks.
- **BMC es un vector de ataque**: el BMC tiene acceso total al hardware (DMA a la RAM del host, control de alimentación). Un BMC comprometido compromete completamente el servidor. Debe estar en una VLAN de management aislada, con firmware actualizado y credenciales fuertes.
