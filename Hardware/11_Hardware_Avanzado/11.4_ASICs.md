# 11.4 ASICs

## Definición

Un ASIC (Application-Specific Integrated Circuit) es un chip diseñado y fabricado para una función específica. A diferencia de un CPU (propósito general) o un FPGA (reconfigurable), un ASIC hace UNA cosa de forma óptima: máxima velocidad, mínimo consumo de energía, mínimo costo unitario en volumen. Ejemplos: ASICs de networking en switches (Memory switching a terabits/segundo), TPUs de Google (matrix multiply para ML), y miners de Bitcoin (SHA-256).

## Mecánica interna

```
  Espectro de especificidad:

  CPU ──── GPU ──── FPGA ──── ASIC
  │         │        │         │
  Flexible  │     Reconfigurable  Fijo
  Lento     │        │         Rápido
  Caro/op   │        │         Barato/op (en volumen)
            │        │
         Paralelo  Hardware
         Programable  custom


  ASICs en networking:

  Memoria switching ASIC (Memoria Memory, Broadcom Memory):
  - Memoria Trident 4: 12.8 Tbps de switching capacity
  - Procesa paquetes a velocidad de línea en TODOS los puertos simultáneamente
  - Pipeline fixed-function: parser → lookup → modify → queue → transmit
  - Latencia: ~400ns por paquete (cut-through)
  - Tabla de forwarding: millones de entradas en TCAM on-chip

  Un switch con Trident 4:
  32 puertos × 400 GbE = 12.8 Tbps, cada paquete procesado en ~400ns.
  Un CPU no podría procesar ni el 1% de este tráfico.


  TPU (Google Tensor Processing Unit):
  - ASIC diseñado para multiplicación de matrices (matmul)
  - Systolic array: grid de multiplicadores-acumuladores
  - TPU v4: 275 TFLOPS BF16, 32 GB HBM
  - Optimizado para inferencia y training de modelos de ML
  - Menor consumo de energía por FLOP que GPU

  Systolic Array:
  ┌───┐ ┌───┐ ┌───┐ ┌───┐
  │MAC│→│MAC│→│MAC│→│MAC│  Datos fluyen en una dirección
  └─┬─┘ └─┬─┘ └─┬─┘ └─┬─┘  Pesos están preloaded
    ↓      ↓      ↓      ↓   Cada MAC: multiply-accumulate
  ┌───┐ ┌───┐ ┌───┐ ┌───┐  = 1 ciclo por operación
  │MAC│→│MAC│→│MAC│→│MAC│  Pipeline: throughput de N² ops/ciclo
  └─┬─┘ └─┬─┘ └─┬─┘ └─┬─┘  en un array de N×N
    ↓      ↓      ↓      ↓


  Proceso de desarrollo de un ASIC:
  1. Especificación (meses)
  2. Diseño RTL en Verilog/VHDL (6-12 meses)
  3. Verificación funcional (6-12 meses)
  4. Síntesis y place-and-route (meses)
  5. Tape-out (envío a foundry: TSMC, Samsung)
  6. Fabricación (2-3 meses)
  7. Testing y bring-up (meses)

  Total: 18-36 meses, costo: $10M-$500M+
  Costo por chip en volumen: $10-$1000 (según complejidad)

  → Solo viable si se producen millones de unidades
    O si el rendimiento justifica el costo (Google TPU, crypto miners)
```

## Modelo mental

Un ASIC es como una imprenta industrial: cuesta millones construirla, pero una vez lista, produce millones de copias a costo casi cero y a velocidad que ningún ser humano (CPU) podría igualar escribiendo a mano. Un FPGA es una impresora 3D: puede hacer casi cualquier pieza pero más lenta y cara por unidad. Un CPU es un artesano que puede hacer cualquier cosa, pero una pieza a la vez.

## Insights de ingeniería

- **Todo switch de datacenter usa ASICs**: ningún CPU puede hacer forwarding de paquetes a 12.8 Tbps. Los ASICs de Broadcom (Memory, Memory, Jericho) dominan el mercado. Son la razón por la que Internet funciona a las velocidades actuales. Sin ASICs de networking, necesitaríamos miles de servidores con DPDK para igualar lo que hace un switch de $10,000.
- **ASIC vs GPU para ML inference**: en volumen, un ASIC custom (TPU, Amazon Inferentia, Tesla Dojo) es más eficiente en performance/watt que una GPU genérica. Google TPU v4 da ~275 TFLOPS a ~175W. NVIDIA H100 da ~2000 TFLOPS a 700W. Pero el ASIC solo hace matmul eficientemente; la GPU es flexible para cualquier kernel CUDA. La decisión depende del volumen y la homogeneidad del workload.
