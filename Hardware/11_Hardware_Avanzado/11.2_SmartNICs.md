# 11.2 SmartNICs

## Definición

Una SmartNIC es una tarjeta de red con capacidad de procesamiento programable integrada (CPU ARM, FPGA, o ASIC) que permite ejecutar funciones de red directamente en la NIC, sin consumir ciclos del CPU del host. Esto incluye firewalling, load balancing, cifrado, encapsulación (VXLAN/Geneve), y procesamiento de storage (NVMe-oF), liberando el CPU del host para workloads de aplicación.

## Mecánica interna

```
  NIC tradicional vs SmartNIC:

  NIC tradicional:                    SmartNIC:
  ┌─────────────────┐                ┌─────────────────────────┐
  │ PHY             │                │ PHY                     │
  │ MAC             │                │ MAC                     │
  │ DMA Engine      │                │ DMA Engine              │
  │ Basic offload   │                │ ┌─────────────────────┐ │
  │ (checksum, TSO) │                │ │ Programmable Engine │ │
  └─────────────────┘                │ │ - ARM cores (8-16)  │ │
                                     │ │ - FPGA fabric       │ │
                                     │ │ - P4 pipeline       │ │
                                     │ │ - eBPF offload      │ │
                                     │ └─────────────────────┘ │
                                     │ eSwitch (hardware)      │
                                     │ Crypto engine (TLS/IPsec│
                                     │ Storage engine (NVMe-oF)│
                                     └─────────────────────────┘

  Ejemplos:
  - NVIDIA BlueField-3: 16× ARM Cortex-A78, 400 GbE, crypto, storage
  - AMD/Pensando Elba: P4-programmable pipeline, 200 GbE
  - Intel IPU (Infrastructure Processing Unit): FPGA + Xeon-D


  Caso de uso: DPU (Data Processing Unit) en cloud:

  Sin SmartNIC:
  CPU del host ejecuta: VM + hypervisor + virtual switch + firewall + encryption
  → 10-30% del CPU consumido en "infraestructura" (overhead tax)

  Con SmartNIC/DPU:
  CPU del host: solo la VM (100% para el cliente)
  SmartNIC ejecuta: hypervisor networking, security, storage offload

  AWS Nitro: exactamente este modelo.
  El Nitro card (SmartNIC custom) maneja:
  - Networking (VPC, Security Groups, EBS networking)
  - Storage (EBS, acceso a discos remotos)
  - Security (Nitro Security Module)
  El CPU EC2 está 100% disponible para el cliente.
```

## Modelo mental

Una NIC tradicional es un buzón pasivo: solo recibe y envía cartas. Una SmartNIC es una oficina de correos integrada en tu edificio: puede clasificar cartas, filtrar spam, cifrar contenido, y reenviar a otros edificios, sin que tú (CPU del host) tengas que hacer nada de eso.

## Insights de ingeniería

- **SmartNICs permiten el modelo "bare-metal as a service"**: en clouds como AWS, el cliente obtiene un servidor físico completo (no VM). La SmartNIC maneja toda la infraestructura de networking y storage de forma transparente. El cliente no sabe que hay una SmartNIC — solo ve una NIC y discos normales.
- **El futuro es mover MÁS funciones a la NIC**: service mesh sidecar (Envoy) consume CPU del host. Ejecutar el sidecar en la SmartNIC elimina ese overhead. NVIDIA BlueField puede ejecutar un Kubernetes node completo en la DPU, separando infraestructura de aplicación a nivel de hardware.
