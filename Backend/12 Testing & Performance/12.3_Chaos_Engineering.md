# 12.3 Chaos Engineering

## Definición

Chaos engineering inyecta fallos controlados en producción (o staging) para validar que los patrones de resiliencia funcionan antes de un incidente real. El principio: si no pruebas tu sistema bajo fallos, no sabes si sobrevive a ellos.

## Mecánica interna

### Tipos de fault injection

| Tipo | Ejemplo | Qué valida |
|------|---------|------------|
| Service failure | Matar un pod/instancia | Auto-recovery, load balancing |
| Latency injection | Agregar 500ms a responses | Timeouts, circuit breakers |
| Network partition | Bloquear tráfico entre servicios | Fallbacks, graceful degradation |
| Resource exhaustion | Limitar CPU/memoria | Throttling, backpressure |
| DNS failure | Bloquear resolución DNS | DNS caching, fallback |
| Disk full | Llenar disco | Log rotation, alerting |

### Herramientas

```
  Chaos Monkey (Netflix): mata instancias aleatoriamente en producción.
  Litmus (CNCF): chaos engineering para Kubernetes.
  Gremlin: plataforma SaaS de fault injection.
  Toxiproxy: proxy que simula condiciones de red adversas.
  tc (traffic control): herramienta Linux para simular latencia/packet loss.
```

### Game Days

```
  Ejercicio estructurado:
  1. Hipótesis: "si Redis se cae, el servicio retorna datos de la DB (más lento pero funcional)"
  2. Inyectar fallo: matar Redis
  3. Observar: ¿el servicio sigue funcionando? ¿alertas se disparan?
  4. Documentar: qué funcionó, qué no, qué mejorar
  5. Remediar: fix los problemas encontrados

  Ejecutar en staging primero, luego en producción (con blast radius controlado).
```

## Insights de ingeniería

- **Start small**: no mates la DB de producción el primer día. Empieza matando una instancia de un servicio stateless en staging.
- **Observabilidad primero**: necesitas logs, métricas y tracing funcionando ANTES de hacer chaos engineering. Si no puedes observar el impacto, no aprendes nada.
- **Chaos in CI/CD**: ejecuta tests de chaos automáticamente en staging después de cada deploy. Si el servicio no sobrevive la muerte de un pod, bloquea la promoción a producción.
