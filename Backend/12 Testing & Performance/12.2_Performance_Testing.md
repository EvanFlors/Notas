# 12.2 Performance Testing

## Definición

Performance testing valida que tu sistema cumple requisitos de rendimiento bajo diferentes condiciones de carga. Incluye load testing (carga esperada), stress testing (punto de quiebre), soak testing (estabilidad), y spike testing (picos).

## Mecánica interna

### Tipos de test

```
  Load Test:   carga esperada → ¿cumple SLOs?
  ─────────────────────────────── throughput constante

  Stress Test: incrementar carga hasta encontrar el límite
  ──────────────────/
                   / ← punto de quiebre
  ────────────────

  Soak Test:   carga normal por horas/días → ¿memory leaks? ¿degradación?
  ──────────────────────────────────────── throughput constante, larga duración

  Spike Test:  pico súbito → ¿se recupera?
  ──────────╱╲──────────── throughput normal con spike
```

### Métricas clave

```
  Latency percentiles:
  p50 = 12ms   ← la mitad de requests tarda menos de 12ms
  p95 = 85ms   ← 95% de requests tarda menos de 85ms
  p99 = 450ms  ← 99% de requests tarda menos de 450ms
  p99.9 = 2s   ← 1 de cada 1000 requests tarda >2s

  Throughput: requests/segundo que el sistema puede manejar.
  Error rate: % de requests con status 5xx.

  Saturation: CPU usage, memory, connection pool, queue depth.
```

### Herramientas

```
  k6 (recomendado):
  - Scripting en JavaScript, extensible
  - CLI, CI-friendly, cloud option

  Locust:
  - Python-based, distributed
  - Web UI para monitoreo en tiempo real

  JMeter:
  - Java, GUI, enterprise
  - Pesado pero completo

  wrk / hey:
  - Simple CLI, HTTP benchmark rápido
  - Para tests rápidos, no escenarios complejos
```

### Ejemplo k6

```javascript
import http from 'k6/http';
import { check, sleep } from 'k6';

export const options = {
  stages: [
    { duration: '1m', target: 100 },   // ramp up
    { duration: '5m', target: 100 },   // sustain
    { duration: '1m', target: 0 },     // ramp down
  ],
  thresholds: {
    http_req_duration: ['p(99)<500'],   // p99 < 500ms
    http_req_failed: ['rate<0.01'],     // <1% errors
  },
};

export default function () {
  const res = http.get('https://api.example.com/users');
  check(res, { 'status 200': (r) => r.status === 200 });
  sleep(1);
}
```

## Insights de ingeniería

- **Test en un ambiente similar a producción**: test en tu laptop no predice rendimiento en producción. Usa el mismo tipo de instancias, red, y datos (o similares).
- **p99 es lo que el usuario siente**: si tu p50 es 10ms pero p99 es 5s, 1 de cada 100 usuarios tiene una experiencia horrible. Optimiza el p99.
- **Soak test para memory leaks**: un servicio puede funcionar perfectamente por 1 hora pero crashear por OOM después de 24 horas (memory leak lento). Soak testing lo detecta.
- **Performance testing en CI/CD**: ejecuta un load test ligero en cada deploy. Si los thresholds fallan (p99 > SLO), bloquea el deploy.
