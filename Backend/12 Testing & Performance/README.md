# Módulo 12 — Testing y Performance

## Propósito

El código sin tests es deuda técnica. El código sin benchmarks es una caja negra. Este módulo cubre la pirámide de testing, performance testing, chaos engineering, y network performance.

## Temas

| Archivo | Tema | Conceptos clave |
|---------|------|-----------------|
| 12.1 | Estrategia de testing | Unit, integration, contract, e2e, testing pyramid |
| 12.2 | Performance testing | Load, stress, soak, spike testing, p50/p95/p99 |
| 12.3 | Chaos engineering | Fault injection, game days, Chaos Monkey |
| 12.4 | Network performance | MSS/MTU, handshake overhead, HOL blocking |

## Dependencias

```
Módulos 1-11 → Módulo 12 (Testing valida todo lo construido)
```
