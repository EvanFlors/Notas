# Módulo 11 — Observabilidad

## Propósito

No puedes arreglar lo que no puedes ver. Los tres pilares de la observabilidad son logs, métricas y traces. Este módulo cubre cómo instrumentar, recopilar, y analizar la salud de tu sistema en producción.

## Temas

| Archivo | Tema | Conceptos clave |
|---------|------|-----------------|
| 11.1 | Logs | Structured logging, correlation IDs, ELK/Loki |
| 11.2 | Métricas | RED/USE methods, Prometheus, SLI/SLO/SLA |
| 11.3 | Distributed Tracing | Spans, traces, OpenTelemetry, sampling |
| 11.4 | Profiling | CPU/memory profiling, flame graphs, continuous profiling |
| 11.5 | Diagnóstico de red | DevTools, mitmproxy, timing breakdown |

## Dependencias

```
Módulo 9 (Arquitectura) → Módulo 11 (Observabilidad del sistema)
```
