# 11.5 Diagnóstico de Red

## Definición

Herramientas para diagnosticar problemas de rendimiento y conectividad a nivel de red: desde DevTools del browser hasta mitmproxy para interceptar HTTPS entre servicios.

## Mecánica interna

### DevTools Networking (timing breakdown)

```
  DNS Lookup:       20ms    ← ¿DNS lento? Cachear, usar DNS local
  TCP Connection:   15ms    ← ¿muchas conexiones nuevas? Keep-alive
  TLS Handshake:    30ms    ← ¿TLS 1.2? Upgrade a 1.3
  TTFB:            200ms    ← ¿backend lento? Profile, optimize query
  Content Download:  10ms    ← ¿payload grande? Compression, pagination
  Total:           275ms

  TTFB (Time To First Byte): lo que más importa para APIs.
  Si TTFB es alto, el problema está en tu backend, no en la red.
```

### mitmproxy

```
  App → [mitmproxy] → Backend service

  Intercepta HTTPS, muestra requests/responses en texto plano.
  Permite: modificar requests, replay, filtrar, scripting.

  mitmproxy         → TUI interactivo
  mitmweb           → Web UI
  mitmdump          → CLI (para scripting y CI)

  Útil para: debugging de APIs de terceros, verificar qué envía tu app,
  simular errores (modificar responses).
```

### Comandos de diagnóstico

```bash
# Timing completo de una request HTTP
curl -w "DNS:%{time_namelookup} TCP:%{time_connect} TLS:%{time_appconnect} TTFB:%{time_starttransfer} Total:%{time_total}\n" -o /dev/null -s https://api.example.com

# DNS resolution
dig +trace api.example.com          # Traza la resolución completa
dig @8.8.8.8 api.example.com       # Consulta DNS específico

# Latencia de red
mtr api.example.com                  # traceroute continuo con estadísticas
ping -c 10 api.example.com          # RTT promedio

# Conexiones activas
ss -tnp | grep :8080                 # Conexiones TCP al puerto 8080
```

## Insights de ingeniería

- **curl -w para diagnosticar TTFB**: si DNS + TCP + TLS suman 50ms pero TTFB es 2s, el problema es tu backend (slow query, downstream lento). Si TLS es 500ms, necesitas TLS 1.3 o connection pooling.
- **mitmproxy para debugging de third-party APIs**: cuando integras con un servicio externo y algo no funciona, mitmproxy te muestra exactamente qué envías y qué recibes, incluyendo headers y encoding.
